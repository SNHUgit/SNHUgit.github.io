# Enhancement-3

[Home](../README.md)

 The artifact I selected for this enhancement was the same CS 320 project I used for the earlier milestones. I chose it again because it gave me a good starting point to keep building on a project I already understood while also showing a clear before and after improvement. The original project used simple in memory storage for contacts, tasks, and appointments, which worked fine for the class it came from but did not feel very realistic for an actual application. For this enhancement, I moved the project to a PostgreSQL database in the Spring Boot version of the application so the data would persist and the project would feel more complete. I also added user authorization, which made the project more realistic by controlling access instead of letting every user reach everything. I think this enhancement fits Course Outcome 4 well because it shows my ability to use practical tools and techniques to build a more useful solution. It also supports Outcome 5 because storing real data and controlling access pushed me to think more carefully about validation, authorization, and protecting information in a way that feels much closer to a real application.

 One thing I learned from this enhancement is that database work changes a project in a different way than the other two milestones did. The software engineering enhancement made the project more usable and professional, and the algorithms enhancement improved search and sorting, but this one made the application feel more permanent and more like a real system. One challenge was keeping this milestone focused on the database side of the project without drifting too far into user interface or algorithm improvements, since by this point the application includes parts of all three categories. Another challenge was making sure the database integration and user authorization worked together cleanly without turning the project into a mess. Overall, I think this enhancement met the goals I planned in Module One because it replaced temporary in memory storage with a real database, improved how the application stores and accesses data, and added a more realistic layer of control over who can use different parts of the system.
