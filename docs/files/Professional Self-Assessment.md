# Professional Self-Assessment

[[Home](../README.md)

 My time in the Computer Science program helped me turn a long interest in programming into something a lot more real and structured. I have been hobby programming for around 20 years, but for most of that time my code was honestly spaghetti code. I could make things work sometimes, but eventually I would always run into an error or design problem that I did not really understand how to solve. That is one of the biggest ways this program helped me. It gave me a stronger foundation for why code works, why it breaks, and how to design it in a way that is easier to understand, maintain, and improve. Building this ePortfolio also helped me see that growth more clearly because it let me look at older work, improve it, and show the difference between where I started and where I am now.

 Some of my biggest aha moments in the program came from learning data structures, Big O notation, and design patterns. Before that, I mostly thought in terms of whether code worked or did not work. After learning those topics, I started thinking more about efficiency, structure, and trade offs. I also had a big mindset change with databases. At one point in my personal life I deleted a database and created a lot of extra work for myself, so ever since then I had kind of walked on eggshells around them. I always wanted to learn them better, but I was ignorant about how to really use them in programming and honestly a little scared of them. Because of that, one of the most valuable parts of this program was learning that databases are not something to avoid, but something to understand and use correctly.

 This portfolio reflects a lot of that growth. I used the same CS 320 artifact across all three enhancement categories so I could turn one simple backend project into a larger and more realistic full stack application. I had some very light experience with Spring Boot before this, but it was more of a brief introduction and more about learning the software security than really understanding how Spring Boot worked. Since I knew I wanted to build a full stack application and felt the most confident with Java, Spring Boot felt like the natural choice. One of the things I am most proud of in this portfolio is that I learned enough Spring Boot to build a beginning to end full stack application with a web interface, application layers, search and sort features, database persistence, data validation and user authorization. I know I have only learned the tip of the iceberg with Spring Boot, but learning a framework like that and using powerful components from a trusted source without trying to reinvent everything gave me a lot more confidence in my ability to become a functional full stack developer.

 The program also helped me better understand communication, collaboration, and professional presentation. Even though a lot of the work was done individually, I still had to learn how to explain technical decisions clearly, accept feedback, and present my work in a way that other people could follow. That matters because software is not just about writing code for yourself. It is about making something that users, stakeholders, teammates, or employers can understand and trust. This capstone especially pushed me to think more about documentation, organization, code comments, cleaner design, and how to explain what I built and why I built it that way. It also helped me think more seriously about security. As my artifact became a real web application with stored data and authorization, it became more obvious that validation, access control, and safer error handling are not optional extras. They are part of building software the right way.

 The artifacts in this portfolio fit together as one story instead of a bunch of disconnected assignments. The original artifact shows where I started with organized classes and service logic. The software engineering enhancement shows how I expanded it into a full stack Spring Boot application. The algorithms and data structures enhancement shows how I improved searching, sorting, and the overall organization of data. The database enhancement shows how I moved the application to PostgreSQL and made it feel more like a real system. Even though this application was designed more for learning than for its specific content, I could very easily change the content and turn it into something more functional, and I already have plans to do that. To me, that says a lot about what I gained from this program. I did not just learn how to complete assignments. I learned how to build a stronger foundation, how to use better tools, and how to turn an idea into something much closer to a real application. That is what I want this portfolio to show. It shows my growth, my direction, and the kind of developer I want to keep becoming.
