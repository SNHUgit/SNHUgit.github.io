# Enhancement-1

[Home](../README.md)

 The artifact I selected was originally created in CS 320, Software Testing, Automation, and Quality Assurance. In its original form, it was a Java project built around contact, task, and appointment classes along with service classes used to add, update, delete, and manage those records. The project was focused much more on validation and test driven development than on user interaction, so it did not include a driver class or any kind of graphical interface. Because of that, it had a solid backend foundation but still left a lot of room for improvement, which made it a good choice to enhance for CS 499.

 I selected this artifact for my ePortfolio because I wanted to use the same project across all three enhancement categories and slowly turn it into one larger and more impressive application instead of just showing a bunch of unrelated assignments. Out of the projects I have completed, this one felt like the best place to start because it already had structured classes, service logic, validation rules, and unit testing, but it was still pretty limited as an actual application. The original model and service classes show my ability to build organized and testable code, while the enhanced version better shows my software design and engineering skills because I turned it into a Spring Boot web application with controllers, Thymeleaf pages, reusable fragments, JavaScript for tab behavior, and centralized exception handling. That improvement took it from being a backend only class project into something that feels a lot more like a real application a person could actually use.

 I think this enhancement met the course outcomes I planned to focus on in Module One, especially Outcomes 2 and 4, and it also made some progress toward Outcomes 1 and 5. My original plan was to redesign the project into a more complete and professional application with a layered structure and a browser based interface, and I think this milestone does that. It supports Outcome 2 by improving how the project is presented visually and technically, and it supports Outcome 4 by showing that I can use tools and techniques like Spring Boot, Thymeleaf, controllers, and organized application layers to build a more realistic solution. I also think it makes some progress toward Outcome 1 because the project is much easier for other people to understand and interact with now than it was in its original form. One update I would make to my original plan is that this enhancement also touched Outcome 5 more than I expected, because validation and exception handling pushed me to think more carefully about bad input, safer design, and how errors should be handled in a cleaner way.

 The biggest thing I learned from enhancing this artifact was that turning a backend only project into a web application is a lot more than just slapping a page on top of some Java classes and calling it good. Thymeleaf was completely new to me, so I had to learn it from scratch, and while it was not horrible to learn, it definitely took time to understand how templates, fragments, and data from controllers all fit together. HTML was honestly probably the hardest part for me because it is not an area I feel very strong in and not one I am especially interested in, so building the user interface took more effort for me than the Java side of the project. Another challenge was keeping the scope under control, because once I got the project working as a web application it became really easy to think of ten more things I could add that were outside the actual assignment. Overall, this process helped me understand software engineering better because it reminded me that building a good application is not just about writing code that works, it is also about structure, usability, maintainability, and knowing when to stop before the project turns into a monster.
